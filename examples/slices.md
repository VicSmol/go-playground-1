## Что такое массивы и срезы

### Массивы (`[n]T`)

Массив — это фиксированная последовательность элементов одного типа. Длина — часть типа.

```go
var arr [3]int = [3]int{1, 2, 3}
```

- Нельзя изменить длину.
- Передаётся по значению (копируется целиком).
- `[3]int` ≠ `[4]int` — разные типы.

### Срезы (`[]T`)

Срез — это динамический, гибкий "окно" на массив. Это **ссылочный тип**.

```go
slice := []int{1, 2, 3}
```

Срез позволяет работать с последовательностями переменной длины. Это наиболее часто используемый тип для коллекций в Go.

## Внутреннее устройство срезов

Срез — это структура из трёх полей (упрощённо):

```go
type slice struct {
    array unsafe.Pointer // указатель на данные
    len   int            // текущая длина
    cap   int            // ёмкость (макс. до которой можно расти)
}
```

- `array` — указывает на базовый массив (на куче или стеке).
- `len` — количество элементов, доступных через `len(slice)`.
- `cap` — количество элементов от `array` до конца базового массива.

> 💡 Срез не владеет данными — он лишь указывает на них.

## Где и как хранятся в памяти

### Массивы

- Хранятся **сплошным блоком** в памяти.
- Расположение (стек/куча) зависит от escape analysis.
- При передаче в функцию — копируются целиком.

### Срезы

- Заголовок среза (структура с `array`, `len`, `cap`) может быть на стеке.
- Данные (базовый массив) — на стеке или куче.
- Если срез "утекает" (escape), данные перемещаются в кучу.

Пример escape:
```go
func getSlice() []int {
    x := make([]int, 3)
    return x // x утекает → данные на куче
}
```

## Как алоцируются, обновляются и очищаются

### Аллокация

- Через литерал: `[]int{1,2,3}`
- Через `make([]T, len, cap)`
- Через `append`

```go
slice := make([]int, 3, 5) // len=3, cap=5
```

При `make`:
- Выделяется блок памяти под `cap` элементов.
- `len` первых элементов инициализируются нулевыми значениями.

### Обновление

- По индексу: `slice[0] = 42`
- Через `append`:

```go
slice = append(slice, 4)
```

Если `len == cap`, происходит **реаллокация**:
1. Выделяется новый массив (обычно в 2 раза больше, если cap < 1024, иначе растёт на 25%).
2. Копируются старые данные.
3. Возвращается новый срез с новым `array` и увеличенным `cap`.

> ⚠️ После `append`, если был reallocation, старый срез указывает на старые данные.

### Очистка

- Срезы не требуют ручной очистки.
- Память освобождается сборщиком мусора, когда на данные никто не ссылается.

Чтобы помочь GC:
```go
slice = slice[:0] // обнуляет len, данные остаются
// или
slice = nil // обнуляет всё
```

Но если другой срез ссылается на те же данные — память не освободится.

## Примеры использования

### Создание
```go
s1 := []int{1, 2, 3}

s2 := make([]string, 0, 10)

s3 := s1[1:3] // срез из s1: [2,3]
```

### Работа с append
```go
s := []int{1}
s = append(s, 2, 3)        // [1,2,3]
s = append(s, []int{4,5}...) // [1,2,3,4,5]
```

### Копирование
```go
dst := make([]int, len(src))
copy(dst, src)
```

`copy` копирует минимум из `len(dst)` и `len(src)`.

## Особенности и поведение

### Срезы передаются по значению

```go
func modify(s []int) {
    s[0] = 999 // изменит оригинальные данные
    s = append(s, 1000) // не повлияет на внешний срез
}
```

- Заголовок копируется, но `array` указывает на те же данные.
- Изменения данных видны снаружи.
- Присваивание `s = append(...)` не влияет на внешний срез.

### Нулевые значения

- `nil`-срез: `var s []int` → `s == nil`, `len(s) == 0`, `cap(s) == 0`
- Пустой срез: `s := []int{}` → `s != nil`, но `len == 0`

Оба работают с `append`, `range`, `copy`.

### Совместное использование данных

```go
s1 := []int{1,2,3,4}
s2 := s1[1:3] // [2,3]
s2[0] = 99
fmt.Println(s1) // [1,99,3,4] — изменился!
```

> ⚠️ Изменения в одном срезе могут повлиять на другой.

## Corner cases

### 1. Реаллокация и потеря данных

```go
s1 := []int{1,2,3}
s2 := s1
s1 = append(s1, 4) // reallocation → s1 указывает на новые данные
s2[0] = 99
fmt.Println(s1) // [1,2,3,4] — не изменился
fmt.Println(s2) // [99,2,3] — изменился
```

### 2. Slice bounds out of range

```go
s := []int{1}
_ = s[10] // panic: runtime error: index out of range
```

### 3. Использование после `nil`

```go
var s []int
s = append(s, 1) // OK
for _, v := range s { ... } // OK
```

Но `s[0] = 1` — panic.

### 4. `len` и `cap` после срезания

```go
s := make([]int, 5, 10) // [0,0,0,0,0], cap=10
s2 := s[2:4]             // len=2, cap=8 (от 2 до конца)
```

## Работа runtime с массивами и срезами

### Создание и управление памятью

- `runtime.makeslice` — выделяет память под срез.
- `runtime.growslice` — увеличивает срез при `append`.
- `runtime.slicebyindex` — реализует оператор `s[i:j]`.

### Сборка мусора

- Сборщик мусора отслеживает указатели на данные срезов.
- Если ни один срез не ссылается на базовый массив — память освобождается.
- Память фрагментируется при большом количестве маленьких срезов.

### Escape analysis

- Компилятор определяет, "утекает" ли переменная за пределы функции.
- Если да — аллоцируется на куче.
- Это позволяет избежать ненужных копий и утечек.

### Оптимизации

- `append` оптимизирован: рост на 25% при больших срезах уменьшает частоту reallocation.
- `copy` интринсифицирован (встроен в код).
- Срезы на стеке не участвуют в GC.

## Заключение

Массивы и срезы — фундаментальные типы в Go. Понимание их внутреннего устройства, особенностей хранения и поведения runtime позволяет писать эффективный, предсказуемый и безопасный код. Срезы, несмотря на простоту интерфейса, требуют внимания к деталям: реаллокации, совместное использование данных, escape analysis и управление памятью.
