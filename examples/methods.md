# Методы в Go

Методы в языке Go — это функции, привязанные к определённому типу. Они позволяют реализовывать поведение для пользовательских типов, что является основой объектно-ориентированного подхода в Go. В отличие от классов в других языках, в Go поведение и данные разделяются: структуры описывают данные, а методы определяют поведение.

## 1. Определение метода

Синтаксис метода похож на функцию, но с добавлением **получателя (receiver)** — специального параметра, указывающего тип, для которого определяется метод.

```go
func (r ReceiverType) MethodName(params) ReturnType {
    // тело метода
}
```

- Получатель указывается в скобках перед именем функции.
- `r` — это имя получателя (похоже на `this` или `self` в других языках).
- `ReceiverType` — тип, для которого определяется метод.

### Пример

```go
type Rectangle struct {
    Width  float64
    Height float64
}

// Метод Area вычисляет площадь прямоугольника
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}
```

Теперь объект типа `Rectangle` может вызывать метод `Area()`:

```go
rect := Rectangle{Width: 5, Height: 4}
fmt.Println(rect.Area()) // 20
```

## 2. Получатель по значению и по ссылке

### Получатель по значению

Когда получатель объявлен как `func (r Rectangle)`, метод получает **копию** экземпляра типа. Это безопасно, но изменения внутри метода не повлияют на оригинальный объект.

```go
func (r Rectangle) SetWidth(w float64) {
    r.Width = w // изменяет только копию
}
```

После вызова `SetWidth` исходный объект не изменится.

### Получатель по ссылке

Чтобы изменить состояние объекта, нужно использовать указатель на тип:

```go
func (r *Rectangle) SetWidth(w float64) {
    r.Width = w // изменяет оригинальный объект
}
```

Теперь вызов `rect.SetWidth(10)` изменит поле `Width` исходного `rect`.

> **Правило хорошего тона:**
> - Если метод **изменяет** состояние объекта — используйте получатель-указатель.
> - Если метод **не изменяет** состояние, но тип **крупный** (struct с большим количеством полей) — используйте указатель для эффективности.
> - Если тип — это встроенный тип (`int`, `string`) или малая структура — можно использовать значение.

## 3. Методы для любых типов

В Go можно определять методы не только для структур, но и для любых именованных типов, определённых в том же пакете.

### Пример с пользовательским типом

```go
type MyInt int

func (mi MyInt) IsEven() bool {
    return mi%2 == 0
}

var x MyInt = 4
fmt.Println(x.IsEven()) // true
```

> ⚠️ **Важно:** Нельзя определять методы для типов из других пакетов (например, `func (s string) Reverse()` — запрещено).

## 4. Методы и интерфейсы

Методы — основа реализации интерфейсов в Go. Интерфейс определяет набор методов, и любой тип, реализующий эти методы, удовлетворяет интерфейсу.

```go
type Shape interface {
    Area() float64
}

// Rectangle реализует интерфейс Shape, потому что имеет метод Area()
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}
```

Теперь `Rectangle` можно использовать везде, где ожидается `Shape`.

## 5. Встраивание (Embedding)

Go не поддерживает наследование, но имеет механизм встраивания, позволяющий включать один тип в другой. При этом методы встроенного типа становятся методами внешнего типа.

```go
type Person struct {
    Name string
}

func (p Person) Greet() {
    fmt.Println("Привет, меня зовут", p.Name)
}

type Employee struct {
    Person  // встраивание
    Company string
}

emp := Employee{
    Person:  Person{Name: "Иван"},
    Company: "Sber",
}

emp.Greet() // вызов метода встроенного типа
```

Если внешний тип определяет метод с тем же именем, он **переопределяет** метод встроенного типа.

## 6. Методы с разными сигнатурами

Go позволяет определять методы с одинаковыми именами для разных типов. Это **не перегрузка** (в Go её нет), а просто разные методы.

```go
type Circle struct {
    Radius float64
}

func (c Circle) Area() float64 {
    return math.Pi * c.Radius * c.Radius
}

// Rectangle также имеет Area() — это нормально
```

## 7. Нулевые значения и методы

Для указателей-получателей важно обрабатывать `nil`:

```go
func (r *Rectangle) IsNil() bool {
    return r == nil
}

var r *Rectangle
fmt.Println(r.IsNil()) // true — безопасно
```

## 8. Методы и пакетная видимость

- Если имя метода начинается с **заглавной буквы** — он экспортируемый (доступен из других пакетов).
- Если с **маленькой** — приватный (только внутри пакета).

```go
func (r Rectangle) Area() float64 { /* экспортируемый */ }
func (r Rectangle) isValid() bool { /* приватный */ }
```

## 9. Методы и функции: в чём разница?

| Метод | Функция |
|-------|--------|
| Привязан к типу | Независима |
| Имеет получателя | Нет получателя |
| Вызывается через `obj.Method()` | Вызывается как `Func(obj)` |

Пример эквивалентной функции:

```go
func Area(r Rectangle) float64 {
    return r.Width * r.Height
}
```

Но метод делает код более выразительным и позволяет использовать интерфейсы.

## 10. Пример: реализация счётчика

```go
type Counter struct {
    value int
}

func (c *Counter) Inc() {
    c.value++
}

func (c *Counter) Dec() {
    c.value--
}

func (c Counter) Value() int {
    return c.value
}

// Использование
var counter Counter
counter.Inc()
counter.Inc()
fmt.Println(counter.Value()) // 2
```

## Заключение

Методы — ключевой механизм инкапсуляции и полиморфизма в Go. Они позволяют:
- Добавлять поведение к типам,
- Реализовывать интерфейсы,
- Строить композицию через встраивание,
- Писать чистый и понятный код.

Go отказывается от классического ООП в пользу композиции и интерфейсов, и методы играют в этом центральную роль.
