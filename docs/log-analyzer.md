# Задача: Реализовать CLI "log-analyzer"

Необходимо создать консольное приложение на Go, которое читает файл логов, агрегирует записи по уровню логирования и компоненту, а затем выводит топ-N самых частых записей.

---

## Описание

CLI "log-analyzer" должен обрабатывать текстовые файлы логов в формате, где каждая строка содержит:
- Временную метку
- Уровень логирования (INFO, WARN, ERROR, DEBUG и т.д.)
- Компонент/модуль (например, "auth", "database", "api")
- Сообщение

Приложение должно:
1. Читать файл логов построчно
2. Парсить каждую строку для извлечения уровня и компонента
3. Агрегировать статистику (подсчитывать количество записей для каждой комбинации уровень-компонент)
4. Сортировать результаты по убыванию количества записей
5. Выводить топ-N результатов (где N задается пользователем)

---

## Требования

1. Приложение должно быть реализовано как CLI-утилита с именем `log-analyzer`
2. Приложение должно принимать два аргумента командной строки:
   - `-file` (обязательный): путь к файлу логов
   - `-top` (опциональный, по умолчанию 10): количество топ-записей для вывода
3. Приложение должно поддерживать форматы логов:
   - `[TIMESTAMP] [LEVEL] [COMPONENT] Message`
   - `TIMESTAMP LEVEL COMPONENT: Message`
   - Другие распространенные форматы (можно начать с одного, затем расширить)
4. Приложение должно агрегировать данные по двум измерениям:
   - Уровень логирования (LEVEL)
   - Компонент (COMPONENT)
5. Приложение должно выводить результаты в читаемом формате:
   - Таблица с колонками: Уровень, Компонент, Количество, Процент
   - Сортировка по убыванию количества
6. Приложение должно обрабатывать ошибки:
   - Файл не существует или недоступен
   - Неверный формат строки лога
   - Пустой файл
7. Приложение должно быть эффективным при работе с большими файлами логов

## Пример использования

### Пример файла логов (`app.log`):
```
2023-10-01T10:30:00 INFO auth User login successful
2023-10-01T10:31:00 ERROR database Connection timeout
2023-10-01T10:32:00 WARN api Slow response detected
2023-10-01T10:33:00 INFO auth User logout
2023-10-01T10:34:00 ERROR database Deadlock detected
2023-10-01T10:35:00 INFO api Request processed
2023-10-01T10:36:00 INFO auth Session created
```

### Запуск приложения:
```bash
# Базовый запуск (top=10 по умолчанию)
./log-analyzer -file app.log

# С указанием количества топ-записей
./log-analyzer -file app.log -top 5

# Вывод справки
./log-analyzer -help
```

### Пример вывода:
```
Топ-5 записей логов:
========================================
Уровень  Компонент  Количество  Процент
----------------------------------------
INFO     auth       3           42.9%
ERROR    database   2           28.6%
INFO     api        1           14.3%
WARN     api        1           14.3%
========================================
Всего записей: 7
```

## Основные компоненты

### 1. Парсер логов (`internal/logparser`)
- **ParseLogLine(line string) (LogEntry, error)** - парсит строку лога
- **LogEntry** структура:
  ```go
  type LogEntry struct {
      Timestamp time.Time
      Level     string
      Component string
      Message   string
  }
  ```

### 2. Агрегатор (`internal/aggregator`)
- **Aggregate(entries []LogEntry) map[string]int** - агрегирует по ключу "уровень-компонент"
- **TopN(aggregated map[string]int, n int) []AggregationResult** - возвращает топ-N записей
- **AggregationResult** структура:
  ```go
  type AggregationResult struct {
      Key       string
      Count     int
      Percentage float64
  }
  ```

### 3. Репортер (`internal/reporter`)
- **PrintTable(results []AggregationResult, total int)** - форматирует вывод в таблицу
- **PrintSummary(total, processed, errors int)** - выводит сводку

### 4. CLI (`cmd/log-analyzer`)
- Обработка аргументов командной строки
- Координация работы парсера, агрегатора и репортера
- Обработка ошибок и вывод справки

## Требования к реализации

1. Использовать только стандартные библиотеки Go (кроме тестирования)
2. Реализовать модульное тестирование для каждого компонента
3. Обеспечить обработку больших файлов через потоковое чтение (буферизация)
4. Поддержать параллельную обработку строк для повышения производительности (опционально)
5. Код должен быть документирован в соответствии с Go conventions
6. Обработка ошибок должна быть явной (не использовать panic)
7. Поддержать возможность расширения для новых форматов логов

---

## Критерии успеха

1. Приложение корректно обрабатывает файлы размером до 1 ГБ
2. Все компоненты покрыты unit-тестами (минимум 80% покрытия)
3. CLI имеет понятный интерфейс и справку
4. Вывод форматируется читаемо и содержит всю необходимую информацию
5. Код хорошо структурирован и следует идиомам Go
6. Обработка ошибок корректна и информативна

---

## Дополнительные возможности (опционально)

1. Поддержка нескольких форматов логов через конфигурацию
2. Вывод в разных форматах (JSON, CSV, таблица)
3. Фильтрация по уровню или компоненту
4. Агрегация по временным интервалам
5. Визуализация статистики (гистограммы, графики)
6. Поддержка чтения из stdin (пайпы)
