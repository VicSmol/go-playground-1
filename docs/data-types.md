## Категории типов данных

Все типы в Go можно разделить на следующие категории:

1. **Основные (примитивные) типы** — встроенные типы, определённые языком.
2. **Составные типы** — типы, построенные из других типов.
3. **Ссылочные типы** — типы, которые ссылаются на данные в памяти.
4. **Интерфейсные типы** — абстракции, определяющие поведение.
5. **Типы-псевдонимы и пользовательские типы** — определяемые разработчиком.

## Перечисление всех типов

Go поддерживает следующие типы:

- **Булевы**: `bool`
- **Числовые**:
  - Целочисленные: `int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `uintptr`
  - Вещественные: `float32`, `float64`
  - Комплексные: `complex64`, `complex128`
- **Строки**: `string`
- **Составные типы**:
  - Массивы: `[n]T`
  - Срезы: `[]T`
  - Карты: `map[K]V`
  - Структуры: `struct{}`
  - Каналы: `chan T`
- **Указатели**: `*T`
- **Функции**: `func(...)`
- **Интерфейсы**: `interface{}` (до Go 1.18), `any` (с Go 1.18+)

> ⚠️ Примечание: `any` — это псевдоним для `interface{}`. В Go 1.26.2 активно используется `any` для лучшей читаемости.

## Основные (примитивные) типы

### Числовые типы

Go предоставляет строгую типизацию для чисел.

#### Целочисленные типы

| Тип      | Размер (бит) | Диапазон значений                    |
|----------|--------------|---------------------------------------|
| `int`    | 32 или 64    | Зависит от архитектуры                |
| `int8`   | 8            | -128 .. 127                           |
| `int16`  | 16           | -32768 .. 32767                       |
| `int32`  | 32           | -2^31 .. 2^31-1                       |
| `int64`  | 64           | -2^63 .. 2^63-1                       |
| `uint`   | 32 или 64    | 0 .. 2^32-1 или 2^64-1                |
| `uint8`  | 8            | 0 .. 255                              |
| `uint16` | 16           | 0 .. 65535                            |
| `uint32` | 32           | 0 .. 2^32-1                           |
| `uint64` | 64           | 0 .. 2^64-1                           |
| `uintptr`| 32 или 64    | Для хранения указателей как числа     |

Пример:
```go
var a int = 42
var b uint8 = 255
var c int64 = 9223372036854775807
```

> 💡 `rune` — это псевдоним для `int32`, используется для представления Unicode-символов.
> `byte` — это псевдоним для `uint8`, используется для байтов.

#### Вещественные типы

| Тип       | Размер | Точность |
|-----------|--------|----------|
| `float32` | 32 бит | ~6-9 дес. знаков |
| `float64` | 64 бит | ~15-17 дес. знаков |

Пример:
```go
var x float32 = 3.14
var y float64 = 2.718281828459045
```

#### Комплексные типы

| Тип         | Компоненты |
|-------------|------------|
| `complex64` | 2x float32 |
| `complex128`| 2x float64 |

Пример:
```go
var z complex128 = 1 + 2i
fmt.Println(real(z), imag(z)) // 1 2
```

### Логический тип

`bool` — принимает значения `true` или `false`.

```go
var flag bool = true
```

> ⚠️ Нет автоматических приведений: `0`, `nil`, `""` не считаются `false` в условиях.

### Строковый тип

`string` — неизменяемая последовательность байтов. Хранит UTF-8 текст.

```go
var s string = "Hello, 世界"
fmt.Println(len(s))        // длина в байтах: 13
fmt.Println(utf8.RuneCountInString(s)) // количество рун: 9
```

Строки можно конкатенировать, но это создаёт новый объект.

```go
s += "!"
```

## Составные типы

### Массивы

Фиксированная длина, тип значения и размер — часть типа.

```go
var arr [3]int = [3]int{1, 2, 3}
```

> ⚠️ `[3]int` и `[4]int` — разные типы.

При передаче в функцию массив копируется целиком.

### Срезы (slices)

Срез — это динамический "окно" на массив. Состоит из:
- Указателя на данные
- Длины (len)
- Ёмкости (cap)

```go
slice := []int{1, 2, 3}
slice = append(slice, 4)
```

Внутренняя структура (упрощённо):
```go
type slice struct {
    array unsafe.Pointer
    len   int
    cap   int
}
```

Срезы передаются по ссылке (копируется заголовок, указывающий на те же данные).

### Карты (maps)

Хеш-таблица `map[K]V`. Неупорядоченная коллекция пар ключ-значение.

```go
m := make(map[string]int)
m["age"] = 30
```

- Не имеет фиксированного размера.
- Ключи должны быть сравнимы (`==`, `!=`).
- `nil`-карта не разрешает запись.

Внутренне — хеш-таблица с масштабированием и обработкой коллизий.

### Каналы (channels)

`chan T` — тип для синхронизации и обмена данными между горутинами.

```go
c := make(chan int, 2) // буферизованный канал
```

- Передача блокирует, если буфер полон (для буферизованных) или приёмник не готов (для unbuffered).
- Закрытие канала: `close(c)`.
- Чтение из закрытого канала возвращает нулевое значение.

### Структуры

Композиция полей.

```go
type Person struct {
    Name string
    Age  int
}

p := Person{Name: "Alice", Age: 25}
```

Поддерживают аноним��ые поля (встраивание):
```go
type Animal struct {
    Species string
}

type Dog struct {
    Animal // встраивание
    Name   string
}
```

### Указатели

`*T` — указывает на значение типа `T`.

```go
var x int = 42
p := &x
fmt.Println(*p) // 42
```

- `&` — адрес значения.
- `*` — разыменование.
- В Go нет арифметики указателей.

### Функции

`func(...)` — тип функции. Можно присваивать, передавать, возвращать.

```go
var f func(int) int = func(x int) int { return x * 2 }
```

## Внутреннее устройство и управление памятью

### Как хранятся типы в памяти

Go использует систему типов времени выполнения (runtime type system). Каждый тип хранится в виде `reflect.Type`, но на низком уровне — через `runtime._type`.

Данные размещаются либо на стеке, либо на куче.

### Стек и куча

- **Стек**: быстрый доступ, автоматическое управление (LIFO), используется для локальных переменных.
- **Куча**: динамическое выделение, управление через сборщик мусора.

Go решает, где хранить переменную, через анализ *escape analysis*.

Пример:
```go
func newInt() *int {
    x := 42
    return &x // x "утекает" в кучу
}
```

### Аллокация и сборка мусора

- Аллокация на куче — быстрая, выполняется параллельно.
- Сборка мусора — трёхцветная, параллельная, потокобезопасная (с Go 1.5+).
- Паузы GC — в диапазоне микросекунд.

Типы ссылаются на свои данные через указатели, если они на куче.

## Особенности типов в Go

- **Строгая типизация**: нет неявных приведений.
- **Нулевые значения**: каждый тип имеет нулевое значение (`0`, `false`, `nil`).
- **Псевдонимы vs. Новые типы**:
  ```go
  type MyInt = int    // псевдоним, совместим с int
  type Age int         // новый тип, несовместим
  ```
- **Сравнимость**: не все типы можно сравнивать. `map`, `slice`, `func` — нельзя сравнивать с `==`, только с `nil`.
- **Упаковка (boxing)**: отсутствует. Интерфейсы хранят тип и данные, но без копирования (если значение помещается в интерфейс).

## Заключение

Go предлагает строгую, безопасную и эффективную систему типов. Понимание внутреннего устройства типов и управления памятью позволяет писать более эффективный и предсказуемый код. Работа с памятью скрыта от программиста, но escape analysis и знание о стеке/куче помогают избегать лишних аллокаций.

Этот конспект охватывает все ключевые аспекты типов данных в Go 1.26.2 и может служить основой для дальнейшего изучения языка.
